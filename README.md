# 投资研究框架与操作手册

**版本**：v1.0 | **更新**：2026-04-01  
**适用范围**：美股+港股+数字资产估值分析、周期拆解、归因模型构建  
**核心原则**：框架 → 因子 → 数据 → 逻辑 → 结论（禁止结论先行）

---

## 一、操作原则（全局约束）

### 1.1 数据诚实度

- **禁止编造**：所有数据必须可验证，无来源即标注不确定性
- **口径一致**：同一指标在不同章节必须使用同一时点、同一定义
- **区分层次**：
  - 🔴 **公开数据**（SEC、财报、政策文件）
  - 🟡 **合理推算**（基于已验证数据的数学推导）
  - 🟢 **分析师判断**（定性观察、经验总结，需明确标注）

### 1.2 推导纪律

```
禁止：结论 ← 找数据
推荐：框架 → 因子 → 数据 → 逻辑 → 结论
```

- **框架优先**：在引入数据之前，先构建因子体系
- **因子无重复**：避免正反馈循环导致的重复计算（见第四章 PYPL 案例）
- **逻辑可追溯**：每条结论都能回溯到具体数据和推导步骤

### 1.3 时间维度划分

投资分析必须区分三个时间尺度及其定价逻辑：

| 时间尺度 | 定价中枢 | 驱动因素 | 分析重点 | 更新频率 |
|---------|--------|--------|--------|---------|
| **长期（1-3年+）** | 基本面/可持续盈利能力 | 商业模式/护城河演变/行业结构 | DCF、可持续增速、终端ROIC | 半年/年度 |
| **中期（3-12个月）** | 周期/估值修复空间 | 增长轨迹、政策、资金配置 | 因子权重、催化剂时序 | 季度 |
| **短期（<3个月）** | 交易情绪/资金节奏 | 消息面、情绪反转、技术面 | 筹码、波动率、资金流向 | 实时 |

**核心纪律**：不将短期交易节奏与长期基本面混淆。

### 1.4 资产分析的五维框架

任何资产的完整分析，必须覆盖以下五个维度，且**先判断哪些维度对该标的最重要**：

#### 维度一：资金面
- **流动性**：日均成交、换手率、流动性溢价
- **筹码结构**：股权结构、大股东锁定期、解禁计划、机构持仓
- **资金渠道**：谁在买？（港股通/QFII/国际配置/散户/产业资本）
- **阶段判断**：筹码集中度变化、成交量与股价的乖离

#### 维度二：结构面
- **商业定位**：真正的利润来源在哪里？（收入vs毛利的结构差异）
- **市场地位**：市占率、增速vs行业、替代品威胁
- **护城河**：
  - 定性描述 ❌
  - 量化退化速度 ✅（如：市占率年均下降 -2ppt）
- **竞争动态**：静态对比表 ❌，动态威胁矩阵 ✅

#### 维度三：制度面
- **监管变量**→**估值参数**的传导（不要停留在"风险披露"）
  - 坏例子：「存在地缘政治风险」
  - 好例子：「若出口管制收紧，主营成本↑3-5%，毛利率→↓200bps，DCF NPV→↓$5/股」
- **政策节奏**：预期变化的时间点与市场反应的历史规律

#### 维度四：情绪面
- **多维交叉**：波动率 + 换手率 + 资金流向 + 搜索趋势
- **避免单一指标**：某一天的成交量爆增不等于转折点
- **情绪反转的量化**：VIX/IV、极端分位数出现的频率

#### 维度五：转折点
- **催化剂必须可操作**：
  - 坏例子：「收入加速」
  - 好例子：「Q2若用户环比+15%以上，则YoY增速从+3%修复至+8%」
- **多重共振**：叙事 + 资金 + 情绪 + 政策的同时出现

## 二、归因模型框架（结构性问题拆解）

### 2.1 什么时候用归因模型

**适用场景**：
- 业绩增速出现异常变化（加速或减速）
- 股票跑输/跑赢大盘或可比公司
- 需要判断变化是**结构性**还是**周期性**

**目的**：
- 将变化拆解到具体因子
- 判断哪些因子**可逆**、哪些**不可逆**
- 估算稳态基本面和估值

### 2.2 五因子框架（PYPL 案例的标准模板）

```
观察现象：Branded Checkout 增速 Q1 2024 +7% → Q4 2025 +1% = 降速 6ppt
```

#### 第一层：大盘 vs 标的相对表现

```
标的增速变化 = 大盘增速变化 + 标的相对表现变化

6ppt 降速 = 1.7ppt（大盘减速）+ 4.3ppt（PayPal 跑输大盘）
```

**数据要求**：
- 大盘数据来自官方机构（Census Bureau、国家统计局等）
- 时点必须对齐（同季同比）
- 无争议的数学运算

#### 第二层：大盘因子（宏观周期）→ 因子 B

**基础贡献**：大盘增速下降 = 1.7ppt（精确）

**溢价贡献**：
- 标的用户群体的宏观敏感性是否 > 大盘平均水平？
- 标的的垂直领域（旅游/游戏/可选消费）是否更受消费放缓冲击？
- 标的的地区结构是否包含高经济风险地区？

**推导方式**：
- 定量锚点：大盘数据（精确）
- 定性调整：管理层表述、用户群体分析、垂直领域研究（估算，需标注）
- **结论形式**：区间，而非点估计

**PYPL 案例结论**：`因子B：1.7-2.5ppt`（下限精确，上限含估算）

#### 第三层：标的特有因素（4.3ppt 跑输）→ 因子 A/C/D/E

需要进一步拆分为**四个互不重复的因子**：

| 因子 | 定义 | 可观测证据 | 推导方式 | 精确度 |
|-----|------|----------|--------|--------|
| **A. 消费者端** | 用户习惯迁移、支付工具偏好改变 | 钱包市占率变化、代际差异、支付方式渗透率 | 市占率流失 × 场景折扣系数 | ⭐⭐ |
| **C. 商户端** | 商户费率敏感性、竞争对手切换、按钮展示度 | 可比商户价格差、平台闭环占比、竞品增速 | 排除法 + 定性证据 | ⭐ |
| **D. 产品过渡** | 新旧产品体验并存期的转化率损失 | 新体验覆盖率、新旧体验增速差 | 加权平均反推 | ⭐⭐ |
| **E. 管理层执行** | 其他可能的主动或被动因素 | 管理层变更、战略调整公告 | 残差 | ❌ |

**关键纪律**：
- 避免 A 和 C 的重复计算（消费者选择↓ → 商户移除按钮 → 消费者更少看到，存在正反馈循环）
- 每个因子必须有**明确的可追溯数据源**或**清晰的推导逻辑**
- 如果无法精确量化，给出区间，而非虚假点估计

### 2.3 归因模型输出的三种形式

#### 形式一：保留区间，承认不确定性（推荐）

> 结构性因子（A+C）合计约 2.5-4.0ppt，周期性+过渡性（B+D）合计约 2.2-4.0ppt。两类因子的贡献在统计意义上**无法严格区分**，但方向判断明确：结构性力量持续不可逆；周期和产品因子有修复空间。稳态增速取决于可逆因子修复幅度——判断为 2-3ppt，对应稳态 3-4%。

**优点**：
- 诚实反映数据限制
- 给出明确的修复假设
- 方便后续动态追踪

#### 形式二：用定序代替伪精度（备选）

> 贡献排序（高→低）：A 消费者迁移 ≥ B 宏观 > C 商户竞争 ≥ D 产品摩擦 >> E 管理层
> 
> **排序依据**：A 有最强独立证据（市占率+代际替换）；B 有最精确量化锚；C 有强定性证据但无法独立量化；D 有可计算逻辑链但关键假设不可验证

**适用场景**：投资者更关心"哪个因子最重要"而非"精确权重"

#### 形式三：只做可精确做的拆分（保守）

> 6ppt = 1.7ppt 大盘减速（精确）+ 4.3ppt PayPal 特有因素（精确）。4.3ppt 的进一步拆分受限于数据，给出方向判断和约束区间，不适合给出百分比权重。

**适用场景**：数据真的无法支撑更细的拆分

### 2.4 常见的归因错误

❌ **错误一**：因子权重加总 = 100%，但各因子相互依赖  
✅ **修正**：标注"粗略拆分，存在重复计算风险"或用排序法

❌ **错误二**：历史对标作为未来增速假设  
✅ **修正**：历史数据是参考，要单独推导"自然增速"假设

❌ **错误三**：一个因子用了多种推导方法，取最符合结论的那个  
✅ **修正**：列出所有可行的推导方式，说明为什么选这个

## 三、研报评估 SOP（七步流程）

### 步骤 0：启动模板（看报告前的尽职调查）

在接触报告内容之前，先确认五个要素：

```
1. 标的：公司名/代码
2. 核心问题：估值？买卖点？行业定位？
3. 读者设定：交易员（1页精华） / 投资者（灰框摘要） / 家办（全文深度）
4. 我已知的疑问：我觉得哪里可能有问题？
5. 材料清单：有哪些参考资料？
```

**输出**：确认清单，为后续评估奠定基线。

---

### 步骤 1：事实核查（先找硬伤，再评逻辑）

**原则**：不评价，只验证。将以下数据逐项搜索求证：

#### 1.1 宏观假设清单

- [ ] 利率、汇率、GDP —— 是否反映最新状态？（超过3个月算过时）
- [ ] 政策预期 —— 研报假设 vs 市场共识是否一致？
- [ ] 行业趋势 —— 引用的数据源时点是什么？

#### 1.2 标的交易结构清单

- [ ] 上市地、交易所、流通盘结构
- [ ] 港股通/QFII 状态、解禁时间表
- [ ] 大股东股权锁定期、减持计划
- [ ] 融资状态、债务结构

#### 1.3 核心财务数据清单

- [ ] 营收、毛利、净利润（与最新财报一致？）
- [ ] 关键运营指标（用户数、ARPU、留存）
- [ ] 一次性项目与持续性利润的分离
- [ ] 各业务线的贡献率与增速

#### 1.4 竞品数据清单

- [ ] 可比公司市值、PE、EV/EBITDA
- [ ] 竞品最新增速、市占率
- [ ] 可比公司的估值是否也处于极端分位（影响对标有效性）

#### 1.5 外部预测清单

- [ ] 卖方共识（来自几家机构？）
- [ ] 分析师目标价（高低点的权重？）
- [ ] 行业预测机构（数据源的可信度？）

#### 1.6 口径一致性检查

```
同一指标在不同章节的数字是否一致？
示例：
- 第2章提到"员工数 5,000"，第5章变成"4,800"？
- 营收在第3章用 RMB，第6章用 USD，换算对吗？
- 市值在第1章用 A 日期，附录用 B 日期，有标注吗？
```

#### 1.7 输出格式

**硬伤清单**（直接影响结论可信度）：
- 数据源过时（超过3个月）
- 基础交易结构假设错误
- 财务数据与公告不符

**软伤清单**（影响专业度但不影响结论方向）：
- 员工数、成本、单位经济参数不统一
- 口径标注不清

**停止点**：如果发现硬伤，呈现给 LEE，等待确认后才进入步骤 2。

---

### 步骤 2：理解标的真实状况（先懂生意，再套框架）

#### 2.1 搜索标的最新动态

- 最近一个季度的经营数据（ARR、用户增长、单位经济）
- 管理层最新表态（Q&A 电话会、媒体访谈）
- 行业内的竞争新闻（竞品发布、并购、政策变化）
- 卖方最新报告的观点变化（升级/下调评级的原因）

#### 2.2 问三个关键问题

**Q1：这家公司靠什么赚钱？**
- 看收入结构，更要看毛利结构
- 高收入的业务线 ≠ 高利润的业务线
- 示例：互联网公司的广告收入占比 vs 广告利润贡献

**Q2：增长靠什么驱动？**
- 真实商业化（付费用户↑、ARPU↑）vs 虚假增长（补贴、会计口径调整、免费用户转化）
- 增速可持续性：是加速还是消费驱动的一次性购买？

**Q3：估值定价的是什么？**
- 当前基本面 vs 远期叙事
- 如果是叙事，核心假设是什么？（用户量 5 倍？毛利率扩张 30ppt？）
- 叙事的实现概率有多大？

#### 2.3 输出格式

```
3-5句话，回答：这家公司的真实状况是什么？市场共识在哪里可能偏离现实？

示例：
「PayPal 的核心利润来自 eCom branded checkout（60% 交易费），而非 P2P（低费率）。
当前增速放缓源于两部分：
①宏观周期（电商大盘放缓）- 可修复
②消费者向 Apple Pay 迁移（长期趋势）- 不可逆

市场可能高估了产品迭代（新 checkout 体验）对消费者习惯的影响。」
```

**停止点**：呈现给 LEE，等确认后进入步骤 3。

---

### 步骤 3：逻辑链审查（框架→因子→数据→结论）

#### 3.1 检查推导顺序

报告是否遵循：**框架 → 因子 → 数据 → 逻辑 → 结论**？

常见违规：
- ❌ 摘要先给评级，正文在"证明"而非"推导"
- ❌ 因子权重是拍脑袋的百分比，无推导过程
- ❌ 只引用支持结论的数据，忽视反面证据
- ❌ 结论改变但框架未更新（追加事件后的被动修改）

✅ 正确做法：
- 先列框架（多少个因子？权重如何设定？）
- 逐个量化因子（数据→推导→区间）
- 加权求和到结论
- 标注哪些假设最敏感，哪些最不确定

#### 3.2 检查归因是否服务投资决策

**坏的归因**：
> 「Q4 增速放缓源于：消费者迁移 42%、宏观周期 25%、商户竞争 20%、产品摩擦 13%。」

然后呢？对持仓有什么含义？

**好的归因**：
> 「增速放缓中：
> - 结构性因素（消费者迁移+商户竞争）= 62%，不可逆
> - 可逆因素（产品摩擦+宏观）= 38%，可修复 2-3ppt
> 
> 推论：即使所有可逆因素修复，稳态增速仍为 3-4%，而市场定价 6% 的增速假设。评级下调。」

#### 3.3 检查估值方法的适用性

| 公司阶段 | 适用方法 | 需要检查什么 |
|---------|--------|-----------|
| **亏损期/高增长** | 长期 DCF | 终端价值占比是否 > 60%？ |
| **成熟期** | PE/PEG、EV/EBITDA | 是否用了正常化盈利？ |
| **周期性** | 周期 PE、可比公司法 | 是否在周期的哪个位置？ |
| **估值锚** | 可比公司 | 可比公司自身估值是否也极端？ |

**常见错误**：
- 用可比公司法，但可比公司都处于估值极端期
- 用 DCF，但终端价值是虚数，占 NPV 的 80%

#### 3.4 输出格式

**逻辑链断裂点清单**：
- [ ] 框架与推导是否一一对应？
- [ ] 哪些结论是演绎（必然），哪些是归纳（概率）？
- [ ] 敏感性分析是否覆盖了关键假设？

---

### 步骤 4：五维深度评估（判断维度优先级，再评估深度）

**关键原则**：不是每个维度都对每个标的同样重要。先判断，再评。

#### 4.1 资金面（谁在交易？流动性够吗？）

**评估项**：
- [ ] 流动性：日均成交 / 股价 = 日均流动性（$）
  - 市值 < $50M：日均成交 < $500K 风险很大
  - 机构配置：需要 $10M+ 的日成交量承载
- [ ] 筹码结构：大股东锁定期、解禁计划
  - 锁定期内：供给受限，可能被高估
  - 临近解禁：供给增加，抛压风险
- [ ] 资金渠道：港股通/QFII/国际配置占比
  - 港股通额度充足 vs 被动限制，意义完全不同
- [ ] 筹码集中度变化趋势

**输出**：流动性充分/不足/缺失，以及对估值的修正

#### 4.2 结构面（生意可持续吗？护城河够深吗？）

**评估项**：
- [ ] 商业定位
  - 毛利贡献 vs 收入贡献：哪个才是真正的核心业务？
  - 单位经济学：CAC / LTV / 转化率 / 续费率
- [ ] 护城河
  - ❌ 定性：「品牌强」「技术领先」
  - ✅ 定量：「市占率 30%，竞品增速 3x，预计 5 年内市占率下降至 20%」
- [ ] 竞争动态：谁在蚕食？增速对比？
- [ ] 如果核心优势是"性价比"而非"不可替代性"，护城河脆弱吗？

**输出**：护城河深度评分 + 退化速度估计

#### 4.3 制度面（政策会改变故事吗？）

**评估项**：
- [ ] 监管变量 → 估值参数的传导
  - ❌ 「存在地缘政治风险」
  - ✅ 「若出口管制，成本↑5%，毛利→↓150bps，DCF 内在价值↓ $8/股」
- [ ] 政策节奏与市场反应的历史规律
- [ ] 如果政策变化是主要风险，给出明确的政策触发条件

**输出**：制度变量对基本面的量化影响

#### 4.4 情绪面（市场温度如何？）

## 四、案例深度拆解：PYPL 归因模型

### 4.1 问题描述

**观察**：PayPal Branded Checkout  
- Q1 2024：YoY +7%  
- Q4 2025：YoY +1%  
- **变化**：降速 6ppt

**目标**：将 6ppt 拆解到五个具体因子，每个因子都要有**可追溯的数学路径**。

### 4.2 第一层拆分：大盘 vs 相对表现

```
标的增速变化 = 大盘增速变化 + 标的相对表现变化

数据：
- 美国电商大盘：Q1 2024 ~7.0% → Q4 2025 +5.3% = -1.7ppt
  （来源：Census Bureau，精确）
- PayPal branded checkout：+7.0% → +1.0% = -6.0ppt
  （来源：SEC earnings，精确）
- PayPal 跑输大盘：0ppt → -4.3ppt = 差值

推导：
6ppt 降速 = 1.7ppt（大盘减速）+ 4.3ppt（PayPal 特有原因）
```

**特点**：完全由官方数据驱动，无争议。

### 4.3 第二层：大盘因子的超额敞口（因子 B）

**问题**：1.7ppt 的大盘减速，对 PayPal 的拖累是否 > 1.7ppt？

**推导过程**：

| 维度 | 观察 | 含义 |
|-----|------|------|
| **用户群体** | PayPal 用户以中等收入为主 | 对消费周期敏感性 > 平均 |
| **垂直领域** | 旅游/Crypto/Gaming（管理层Q4 call） | 可选消费，景气度下行首当其冲 |
| **地理结构** | 德国、欧洲占国际收入 30%+，经济疲软 | 区域特有的宏观负面 |
| **结论** | PayPal 宏观敞口 > 大盘平均 | 拖累幅度 > 1.7ppt |

**不确定性处理**：

- **精确部分**：1.7ppt（大盘官方数据）
- **估算部分**：超额敞口约 +0.8-1.0ppt（定性判断，基于用户群特征）
- **结论区间**：因子 B = 1.7-2.5ppt

**关键方法**：
- 定量锚点 + 定性调整
- 不强行给出点估计，而是区间

### 4.4 第三层：标的特有的 4.3ppt（因子 A/C/D/E）

这是最难的部分，因为 4.3ppt 需要在四个互不重复的因子之间分配。

#### 因子 A：消费者习惯迁移

**可观测证据**：
- Bernstein：PayPal 数字钱包份额 45% (2023) → 40% (2025) = -5ppt
- Apple Pay：全球在线份额 8% (2024) → 14.2% (2025)，增速 +77%
- 代际差异：89% Boomers 用 PayPal vs 74% Gen Z；50% Gen Z 用 Apple Pay vs 15% Boomers

**问题**：
- 钱包份额 -5ppt 是**全场景**指标（含 P2P、unbranded、线下）
- 但 branded checkout 的份额流失不能直接观测

**数学路径**：

```
如果：branded TPV = 电商大盘 × branded 渗透率

则：branded 增速 ≈ 电商增速 + 渗透率变动率

观察数据：
- 电商增速：+5.3%
- branded 增速：+1.0%
- 推导：渗透率变动率 = 1.0% - 5.3% = -4.3%

（即：branded 渗透率年化下降 4.3%）
```

**间接约束**：

- 全场景钱包份额年化流失：-5/45 ≈ -11.1%
- branded 渗透率仅下降 -4.3%，远小于 -11.1%
- 差异来自：
  - ① 全场景包含 unbranded/P2P（与 branded 无关）
  - ② branded 场景的转换成本高（商户集成）
  - ③ BNPL + Venmo Pay With 部分对冲流失

**因子 A 归因**：

假设 branded 场景的转换成本折扣系数 = 40-60%，则：

```
因子 A ≈ 全场景份额流失速度 × 折扣系数
       ≈ (-5ppt / 45% × 变动率) × 40-60%
       ≈ 1.5-2.5ppt
```

**诚实标注**：40-60% 的折扣系数是定性判断，无精确数据支撑。

#### 因子 C：商户侧竞争与费率

**可观测证据**：
- Shop Pay (Shopify) 单季 GMV +50%
- 费率差：PayPal 2.99%+$0.49 vs Shopify 1.6%+$0.30
- 平台闭环（Amazon+Walmart+Shopify）占美国电商 55%
- PayPal 2024 年逆向提价：2.6% → 2.99%
- Bandcamp 从 PayPal 全面迁移至 Stripe

**传导渠道**：
1. 按钮消失：商户移除 PayPal 按钮 → 消费者无选择 → TPV 直接损失
2. 按钮降级：PayPal 从首选降至次选 → 转化率下降

**问题**：两个渠道都**无法直接量化**（PayPal 不披露按钮展示数据）

**推导方法**：排除法

```
4.3ppt 总跑输 = 因子 A + 因子 C + 因子 D + 因子 E

已知：
- 因子 A = 1.5-2.5ppt
- 因子 D = 0.5-1.5ppt（见下文）
- 因子 E = 残差

则：因子 C = 4.3 - (A+D) = 1.0-1.5ppt
```

**特点**：无法独立量化，只能通过排除法约束。

#### 因子 D：产品迁移摩擦

**可观测证据**：
- 新 checkout 体验覆盖率：年初 5% → 目前 25%
- 部署完整 playbook 的商户在 Cyber Five 期间 branded TPV **双位数增长**（管理层 Q4 call）
- 生物识别注册：月增 30 万（FIDO Alliance）

**这是唯一可以正向推导的因子**。

**推导过程**：

```
设：
- 新体验商户增速 = X%（"双位数" = 10-15%，取 +12%）
- 旧体验商户增速 = Y%
- 新体验覆盖率 = 25%
- 旧体验覆盖率 = 75%

则：整体 branded 增速 = 25% × X + 75% × Y = 1%

代入 X = 12%：
25% × 12% + 75% × Y = 1%
3% + 75%Y = 1%
Y = -2.7%

推论：
旧体验商户的 branded checkout 实际在萎缩 -2.7%
新体验商户在增长 +12%

迁移摩擦的贡献 = 差值造成的 TPV 损失
```

**保守估算**：

```
如果旧体验的"自然"增速（无摩擦）是 0%（即与大盘持平），
而实际是 -2.7%，
则迁移摩擦 = -2.7ppt × 75% 覆盖率 = -2.0ppt

但这个假设（旧体验"自然增速=0"）无法验证。

更保守的范围：0.5-1.5ppt
```

**特点**：逻辑链完整，但关键假设（旧体验的"自然"增速）无法验证。

#### 因子 E：管理层执行力（定义为残差）

```
因子 E = 4.3ppt - (A + C + D)
       = 4.3 - (1.5-2.5 + 1.0-1.5 + 0.5-1.5)
       = 4.3 - (3.0-5.5)
       = -1.2 到 +1.3ppt

残差包含零，说明：
1. 管理层因素确实很小
2. 或前三个因子的估算偏高

因子 E 归因：0-0.5ppt（残差区间的保守端）
```

### 4.5 完整归因总结

| 因子 | 区间 | 占比 | 推导方法 | 精确度 |
|-----|------|------|---------|--------|
| **B. 宏观周期** | 1.7-2.5 | 28-42% | 大盘官方数据+用户群敏感性 | ⭐⭐⭐ |
| **A. 消费者迁移** | 1.5-2.5 | 25-42% | 全场景份额流失×折扣系数 | ⭐⭐ |
| **C. 商户竞争/费率** | 1.0-1.5 | 17-25% | 排除法+定性证据 | ⭐ |
| **D. 产品摩擦** | 0.5-1.5 | 8-25% | 新旧体验增速差 | ⭐⭐ |
| **E. 管理层** | 0-0.5 | 0-8% | 残差 | ❌ |
| **合计** | 5.2-8.0 | — | — | 区间较宽 |

**⚠️ 诚实的声明**：
- 区间合计 5.2-8.0，覆盖了 6ppt 但宽度较大（±1.5ppt）
- 原因：因子 A 和 C 存在正反馈循环（消费者↓ → 商户移除按钮 → 消费者更少看到），简单加总有重复计算风险
- 强制收窄到点估计（比如都取中值 = 6.0）是**虚假精度**

### 4.6 投资结论的表述

**表述方式**（保留区间，承认不确定性）：

> 结构性因子（A+C）= 2.5-4.0ppt，周期性+过渡性（B+D）= 2.2-4.0ppt。
>
> 两类因子的相对贡献在统计意义上**无法严格区分**，但方向判断明确：
> - 结构性力量（消费者习惯+商户竞争）是**持续、不可逆**的
> - 周期和产品因子有**修复空间**
>
> 稳态增速取决于可逆因子能修复多少。我们的判断是 2-3ppt，对应**稳态 branded 增速 3-4%**。
>
> 市场当前定价 6% 增速，意味着定价了 3-4ppt 的修复空间。这个修复发生的概率和时间框架，是估值重估的关键。

**区间 vs 点估计的区别**：
- ❌ 「稳态增速 3.5%」（虚假精度）
- ✅ 「稳态增速 3-4%」（诚实反映不确定性，同时给出可操作的范围）

## 五、常见陷阱与纪律

### 5.1 归因陷阱

| 陷阱 | 表现 | 修正 |
|-----|------|------|
| **因子重复** | A 和 C 的正反馈循环导致 1+1>2 | 标注"存在交互，简单加总有偏差" |
| **历史外推** | 用历史对标做未来假设 | 历史是参考，需单独推导自然增速 |
| **多元归因** | 用多种推导方法，选最符合结论的 | 列出所有可行方法，说明选择理由 |
| **点估计迷思** | 1.5ppt + 1.2ppt + 0.8ppt = 3.5ppt | 给区间，标注不确定性 |

### 5.2 估值陷阱

| 陷阱 | 表现 | 修正 |
|-----|------|------|
| **终端价值过高** | DCF 中终端价值 > 70% NPV | 做敏感性分析，说明对终端增速假设的依赖度 |
| **可比公司极端** | 用估值极端期的可比公司做锚 | 用历史中位数或正常化盈利做锚 |
| **遗漏真实成本** | 只看 EBITDA，忽视 CAPEX/WC | 做自由现金流分析 |
| **混淆时间维度** | 短期交易价格作为长期定价依据 | 明确评估的时间框架 |

### 5.3 数据陷阱

| 陷阱 | 表现 | 修正 |
|-----|------|------|
| **过时数据** | 引用 6 个月前的宏观数据 | 宏观数据每月更新，及时补充最新值 |
| **口径混用** | 营收用 GAAP，对比时用非 GAAP | 始终保持同一口径，重新计算对比 |
| **样本偏差** | 可比公司样本只有 3 家 | 扩大样本到 5+ 家，看中位数而非平均 |
| **单一来源** | 某个指标只引用 1 个卖方报告 | 用 2+ 个来源交叉验证 |

### 5.4 逻辑陷阱

| 陷阱 | 表现 | 修正 |
|-----|------|------|
| **因果混淆** | 相关性当因果性 | 用时间序列看先后，用对照组看独立性 |
| **幸存者偏差** | 只看成功的案例 | 同时看失败的案例和失败的类似情况 |
| **确认偏误** | 只收集支持结论的证据 | 主动寻找反面证据，评估其权重 |
| **归纳陷阱** | 从 3 个案例归纳出普遍规律 | 说"可能"而非"一定" |

## 七、高阶技巧

### 7.1 构建可复用的估值模型

**目标**：同一套模型，换标的就能用

**四步法**：

1. **参数化**：将所有关键假设提取为可调节的参数
   - 收入增速、毛利率、税率、折现率、终端增速
   
2. **敏感性网格**：二维（最常见）或三维敏感性表
   - 横轴：参数 A（如收入增速）
   - 纵轴：参数 B（如毛利率）
   - 单元格：估值
   
3. **情景分析**：至少 3 个情景（基础/乐观/悲观）
   - 各情景的参数组合逻辑要可讲述
   
4. **概率加权**：给每个情景赋予概率
   - 公开信息推导而来，不是拍脑袋

### 7.2 识别市场共识与非共识

**方法**：
1. 搜集至少 5 家卖方的目标价与假设
2. 找到共识假设的中位数
3. 你的假设与共识的偏差在哪里？
4. 偏差有多大？偏差的论证有多强？

**示例**：
```
卖方共识：PYPL 2024-2026 年 EBITDA 增速 +4% CAGR
我的预测：+2% CAGR

差异来自：
- 共识假设 branded checkout 稳态增速 6%
- 我的预测：3-4%（基于归因模型）

论证强度：⭐⭐⭐（有数据支撑的归因模型）
市场认可度：低（多数分析师尚未做这个拆解）

→ 如果我的预测被验证，股价有向下修正空间 15-20%
```

### 7.3 动态跟踪框架

**每个季度更新**：

```
| 假设 | Q3 预测 | Q4 实际 | 偏差 | 根本原因 | 框架调整 |
|-----|--------|--------|------|---------|---------|
| 电商增速 | +5.5% | +5.2% | -0.3% | 消费走弱 | 继续下调 |
| branded 渗透率 | -3.5% | -4.1% | -0.6% | 消费者迁移超预期 | 因子A调升 |
| 新体验覆盖率 | 22% | 25% | +3% | 商户采纳快 | 产品摩擦缩小 |
```

每个偏差背后都有**根本原因**，而根本原因会触发**框架调整**。

### 8.3 常见问题速查

**Q：什么时候给区间，什么时候给点估计？**  
A：
- 有充足数据支撑 → 点估计 (同时给±敏感范围)
- 数据有限但逻辑清晰 → 区间 (标注上下限依据)
- 数据不足+逻辑不确定 → 定序法 (高/中/低，不给数字)

**Q：如何判断我的框架是否完整？**  
A：
- 用排除法：如果漏掉一个因子，结论是否会改变？
- 若漏掉会改变 → 需要加入这个因子
- 若漏掉不改变 → 这个因子可能真的不重要（标注）

**Q：何时应该更新研报？**  
A：
- 硬触发：新财报发布、政策变化、重大并购
- 软触发：框架关键参数偏差 > 20%、市场共识明显移动
- 定期触发：半年度全面复盘

**最后的话**

这套框架的目标不是给出精确的答案，而是：

1. **透明化推导过程**，让每个结论都可追溯
2. **诚实标注不确定性**，不为了精度而编造数据
3. **让框架可复用**，换个标的就能套用
4. **识别可逆 vs 不可逆因素**，用于投资决策

执行时的铁律：**逻辑 > 数据 > 可验证性 > 表达**

没有完美的研报，只有诚实的研报。
